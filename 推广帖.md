## 开源分享

前段时间捣鼓了个小工具，叫 PicaScraper。

说白了就一件事——你登一下哔咔账号，它帮你把收藏夹里的漫画全扒下来存到本地，然后你可以像本地阅读器一样翻、搜、分类、导出，甚至还能拿到排行榜上挑新漫画一键收藏。

跟那种"开个浏览器手动一话一话另存为"的体验完全不一样，它直接调 API，全自动跑。

支持干的事情：

- 同步收藏夹（734 部漫画一次性拉下来）

- 批量下载（漫画级、章节级、图片级三层并发，速度可调）

- 本地浏览（网格视图、详情页、分类筛选、分页）

- 内置阅读器（瀑布流滚动 + 翻页模式，支持专注模式）

- 断点续传（两层进度追踪，断了重跑不重复下载）

- 三源搜索（收藏夹 API / 本地已下载 / 导入漫画）

- 排行榜浏览（24h、周榜、月榜，一键收藏到账号）

- 批量导出/导入（ZIP 打包，迁移备份方便）

- 模糊模式（公共场合浏览自动打码，应急必备）

- 暗色主题 / 哔咔原版配色，二选一

- 类似漫画推荐（基于语义向量，会把描述相近的漫画推给你，可关闭）

我的使用场景大概是：

睡前批量挂着下载几百话第二天通勤路上看，整理本地库存避免哪天版权变动东西没了，导出一批本子给平板离线看，排行榜上发现新漫画顺手收藏然后挂下载。

做这个东西的原因也挺简单——哔咔网页版不能批量下载，APP 又不能在电脑上看，收藏夹 700 多部漫画想本地存一份根本没趁手工具。所以就自己逆向了 API 签名算法，搭了个全套。

现在这版算是能用了，但绝对不是那种"看起来像正式版挂着 beta 名号"的东西，是真实诚的还在迭代。

已知问题：

- Token 每 7 天左右过期，目前不会自动刷新，得手动重新登录一下
- 并发调太高容易被限流，建议图片并发 5~8 就够了，别拉满 80
- 向量推荐依赖阿里云 DashScope API，需要自己搞 Key（不用也不影响核心功能）
- 目前只适配了哔咔，没有做多站适配器
- 下载状态是单进程的，别同时开两个浏览器下载
- 不提供打包好的 exe，需要本地有 Python 环境

怎么跑起来：

1. `git clone` 下来
2. 双击 `setup.bat`（Windows）或 `bash setup.sh`（Mac/Linux）
3. 去哔咔网页版 https://manhuapica.com 登录，F12 → Application → Local Storage，复制 `token` 和 `nonce` 的值
4. 打开自动生成的 `config.yaml`，把 token 和 nonce 填进去
5. `python server.py`
6. 浏览器打开 `http://localhost:8000`
7. 先去收藏页点"刷新"同步收藏夹，然后去下载页勾选漫画开始下载

技术栈：Python + FastAPI + httpx(asyncio) + 原生 HTML/CSS/JS（零前端框架），前端纯手搓 SPA。

**关于 vibecoding**

因为是第一次用 Python，这个项目从头到尾 99% 的代码都是对着 AI 一句一句问出来的，我做的事基本就是：描述需求 → 看 AI 给方案 → 跑起来 → 报错 → 把报错贴回去 → 修好 → 继续下一步。循环了不知道多少轮。

最大的感受是——以前觉得"不会编程就做不了软件"，现在发现只要你能把需求说清楚、有耐心 debug、愿意读报错信息，AI 真的能帮你把东西怼出来。中间踩过最大的坑是对着 AI 要了一份异步代码，但自己完全不懂 asyncio 是什么，出了 bug 连日志都看不懂，只能把整个 traceback 丢回去让它自己修。

逆向 API 签名那块最折磨人。哔咔的 JS 混淆了好几层，光是定位 sign 函数就在浏览器 DevTools 里蹲了一晚上。但也是整个项目里最有成就感的部分——从一团乱码到 HMAC-SHA256 最后算出正确的签名打回 API，那种感觉跟解谜一样。

总之就是一句话：AI 降低了"从 0 到 1"的门槛，但"从 1 到能跑"还是得靠你自己有耐心磨。如果你也想 vibecoding，建议从一个你自己真的会用的东西开始——有真实需求撑着，遇到 bug 才没那么容易放弃。

想收集的反馈：

- 收藏同步正不正常？
- 下载断点续传有没有翻车？
- 阅读器体验怎么样？瀑布流和翻页哪个舒服？
- 向量推荐推出来的东西靠不靠谱？
- 配置页面好不好理解？
- 有遇到什么反爬/限流/报错吗？
- 哪里你觉得用着别扭？

大概就是这么个东西，不完美但至少能用。有空的兄弟试试，先谢谢了。
