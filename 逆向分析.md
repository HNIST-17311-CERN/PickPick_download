### 3.3 从请求头推断签名结构

对比两组数据：

| 字段 | 第1页 | 第2页 | 是否变化 |
|------|-------|-------|----------|
| URL | `page=1` | `page=2` | 是 |
| time | `1781513324` | `1781514033` | 是 |
| signature | `c413bfb6...` | `8bbf363c...` | 是 |
| nonce | 不变 | 不变 | 否 |
| authorization | 不变 | 不变 | 否 |

**推断**：签名至少与 `path + time` 相关。nonce 和 token 不变，但签名变了 → nonce 和 token 不是签名的唯一输入。

signature 是 64 字符 hex = 256 bits = SHA-256，大概率是 **HMAC-SHA256**。

## 5. 还原混淆算法

### 5.1 Ue 函数分析

`Ue()` 用于解码 JS 中存储的混淆字符串（即密钥）：

```javascript
function Ue(e) {
    try {
        const a = Oe(atob(e), "PicaWeb2025");  // Step 1+2: base64解码 → Fisher-Yates Deshuffle
        const o = a.split("").map(e =>
            String.fromCharCode(e.charCodeAt(0) ^ 42)  // Step 3: XOR 42
        ).join("");
        return atob(o);  // Step 4: base64解码
    } catch (a) {
        return "";
    }
}
```

### 5.2 Oe 函数分析（Fisher-Yates Deshuffle）

```javascript
function Oe(e, t) {
    // 1. 创建索引数组 [0, 1, 2, ..., n-1]
    const a = Array.from({length: e}, (_, i) => i);
    
    // 2. 用种子字符串的字符码之和初始化随机数
    let o = 0;
    for (let r = 0; r < t.length; r++)
        o += t.charCodeAt(r);
    
    // 3. Fisher-Yates shuffle（用 LCG 生成伪随机数）
    for (let r = a.length - 1; r > 0; r--) {
        o = (9301 * o + 49297) % 233280;
        const e = o % (r + 1);
        [a[r], a[e]] = [a[e], a[r]];
    }
    
    // 4. 用打乱后的索引重排输入字符串
    const s = e.split("");
    const r = new Array(e.length);
    for (let n = 0; n < s.length; n++)
        r[n] = s[a[n]];  // 第n个位置 = 输入的第a[n]个字符
    
    return r.join("");
}
```

**关键理解**：`Oe` 不是加密，而是**确定性重排**——同一个种子 `"PicaWeb2025"` 总产生相同的 shuffle 序列，因此 `Ue` 是可逆的。

### 5.3 Python 复现

```python
def _oe_decode(s: str, seed: str) -> str:
    """Fisher-Yates deshuffle — 逐行翻译 JS Oe 函数"""
    n = len(s)
    arr = list(range(n))

    # 种子求和
    t = sum(ord(c) for c in seed)

    # Fisher-Yates shuffle (LCG: o = (9301*o + 49297) % 233280)
    for i in range(n - 1, 0, -1):
        t = (9301 * t + 49297) % 233280
        j = t % (i + 1)
        arr[i], arr[j] = arr[j], arr[i]

    # 重排：result[i] = input[arr[i]]
    result = [''] * n
    for i in range(n):
        result[i] = s[arr[i]]
    return ''.join(result)


def _ue(encoded: str) -> str:
    """解码混淆字符串 — 逐行翻译 JS Ue 函数"""
    step1 = base64.b64decode(encoded).decode('latin-1')
    step2 = _oe_decode(step1, 'PicaWeb2025')
    step3 = ''.join(chr(ord(c) ^ 42) for c in step2)
    step4 = base64.b64decode(step3).decode('latin-1')
    return step4
```

## 6. 还原签名算法

### 6.1 签名计算流程

从 `He` 函数还原：

```
输入:
  e = 完整请求 URL（如 "https://picaapi.go2778.com//users/favourite?page=1&s=dd&limit=20"）
  t = Unix 时间戳（如 "1781513324"）
  o = HTTP 方法（如 "GET"）

Step 1: 去掉 URL 的 base 部分
  path = e.replace("https://picaapi.go2778.com/", "")
       → "/users/favourite?page=1&s=dd&limit=20"

Step 2: 拼接消息
  message = path + t + nonce + o + secret1
          = "/users/favourite?page=1&s=dd&limit=20" + "1781513324" + "7nhqmczfn78mpxmwmraxdckjdzmh4k67" + "GET" + "C69BAF41DA5ABD1FFEDC6D2FEA56B"

Step 3: 去掉 path 开头的 "/"
          = "users/favourite?page=1&s=dd&limit=20" + "1781513324" + "7nhqmczfn78mpxmwmraxdckjdzmh4k67" + "GET" + "C69BAF41DA5ABD1FFEDC6D2FEA56B"

Step 4: 转小写
          = "users/favourite?page=1&s=dd&limit=2017815133247nhqmczfn78mpxmwmraxdckjdzmh4k67getc69baf41da5abd1ffedc6d2fea56b"

Step 5: HMAC-SHA256
  signature = HMAC-SHA256(secret2, message)
```

### 6.2 HMAC 实现确认

JS 中的 `function(e, t)` 是标准 HMAC-SHA256 的手动实现：

```javascript
const o = 64;           // SHA-256 block size
const r = new TextEncoder;
let n = r.encode(t);   // t = 密钥

// 密钥填充/截断到 64 字节
if (n.length > o) n = SHA256(n);  // 长密钥先 hash
if (n.length < o) {               // 短密钥补零
    const e = new Uint8Array(o);
    e.set(n);
    n = e;
}

// ipad = 密钥 XOR 0x36, opad = 密钥 XOR 0x5C
const ipad = new Uint8Array(o);  // 54 = 0x36
const opad = new Uint8Array(o);  // 92 = 0x5C
for (let a = 0; a < o; a++)
    ipad[a] = 54 ^ n[a],
    opad[a] = 92 ^ n[a];

// HMAC = SHA256(opad || SHA256(ipad || message))
const inner = SHA256(ipad + message);
const outer = SHA256(opad + inner);
return hex(outer);
```

这是**标准 HMAC-SHA256**，Python 可以直接用 `hmac.new(key, msg, hashlib.sha256)`。

### 6.3 Python 完整实现

```python
def compute_signature(path: str, timestamp: str, nonce: str, method: str) -> str:
    p = path.lstrip('/')
    msg = (p + timestamp + nonce + method + SECRET1).lower()
    return hmac.new(SECRET2.encode(), msg.encode(), hashlib.sha256).hexdigest()
```

## 9. 关键发现

1. **秘密藏在哪里**：两个密钥通过 `Ue()` 混淆后硬编码在 `shared-core-3jdD82dM.js` 中，浏览器端直接可读
2. **混淆不是加密**：`Ue()` 只是 base64 + 重排 + XOR 的编码变换，不依赖外部密钥，任何拿到 JS 文件的人都可以还原
3. **Web 版与 App 共用 API Key**：`secret1 = C69BAF41DA5ABD1FFEDC6D2FEA56B` 与网上公开的 Android App API Key 一致
4. **Web 版额外有一个 HMAC 密钥**：`secret2` 是一个 64 字节的随机字符串，用于 HMAC-SHA256 签名，App 版可能没有这一层
5. **API 路由**：Web 版 API 在 `picaapi.go2778.com`（不是 `acbbb.com` 或 `picacomic.com`），路径格式与 App API 相似但通过不同的网关
