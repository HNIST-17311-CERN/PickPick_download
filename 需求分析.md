# 漫画爬虫脚本 - 需求分析文档

## 1. 项目概述

### 1.1 项目目标
开发一个 Python 漫画爬虫脚本，支持从漫画网站批量爬取漫画，按章节组织保存为图片文件夹。

### 1.2 核心定位
- 通用可配置框架，通过适配层对接不同漫画网站
- 支持多种爬取模式：全站、按分类/标签、按排行/更新
- 本地图片文件夹输出，按 `漫画名/章节名/` 层级组织

---

## 2. 功能需求

### 2.1 爬取模式

| 模式 | 说明 |
|------|------|
| **全站爬取** | 遍历网站上所有漫画，逐个爬取所有章节 |
| **按分类/标签** | 指定分类或标签，只爬取符合条件的漫画 |
| **按排行/更新** | 按排行榜或最近更新列表爬取 Top N |
| **指定漫画** | 手动指定一部或多部漫画的 URL 列表 |

### 2.2 漫画元数据获取
- 漫画名称、作者、封面图（可选下载）
- 分类/标签信息
- 最新章节、更新时间
- 章节列表（编号、标题、页数、发布时间）

### 2.3 章节图片下载
- 逐页下载章节内所有图片
- 支持断点续传（已下载的图片自动跳过）
- 图片格式：保持原始格式（jpg/png/webp/avif）
- 文件命名：按页码补零对齐（001.jpg, 002.jpg, ...）

### 2.4 输出目录结构
```
downloads/
└── 漫画名/
    ├── metadata.json          # 漫画元信息
    ├── cover.jpg              # 封面图（可选）
    ├── 第001话-标题/
    │   ├── 001.jpg
    │   ├── 002.jpg
    │   └── ...
    └── 第002话-标题/
        └── ...
```

### 2.5 进度与状态管理
- 下载进度实时显示（当前漫画/章节/页码）
- SQLite 本地数据库记录已下载任务，支持断点续传
- 下载失败自动重试（可配置重试次数和间隔）
- 日志文件记录运行详情和错误信息

### 2.6 认证与 Cookie
网站需要登录才能访问漫画内容，脚本不实现登录逻辑，而是通过用户手动提供的 Cookie 维持登录态。

**Cookie 提供方式（按优先级）：**
1. 配置文件 `config.yaml` 中直接填写 `cookie` 字段
2. 命令行参数 `--cookie "key1=val1; key2=val2; ..."`
3. 从浏览器本地 Cookie 文件自动提取（`browser-cookie3` 库，可选）

**Cookie 管理：**
- 脚本启动时自动校验 Cookie 有效性（发一个测试请求确认登录态）
- Cookie 失效时明确提示用户更新，不静默失败
- 支持多组 Cookie 轮换（多账号），降低单账号请求频率

### 2.7 运行控制
- 并发下载数可配置（章节级并发 + 页级并发）
- 请求间隔控制，避免被封 IP
- 支持代理配置
- 支持限速（每秒最大请求数）

## 3. 非功能需求

### 3.1 性能
- 支持并发下载，默认 3 章节并行，每章节 3 页并行
- 图片下载使用流式写入，避免大图占用内存
- 数据库批量写入减少 IO

### 3.2 可靠性
- 每个任务状态持久化到 SQLite，中断后可恢复
- 图片下载校验（文件大小 > 0，可选 hash 校验）
- 失败自动重试 3 次，指数退避

### 3.3 可维护性
- 网站适配层与核心引擎分离（策略模式）
- 配置文件驱动（YAML/JSON），弱化代码与配置的耦合
- 清晰的模块划分和日志输出

### 3.4 合规性
- 遵守 robots.txt
- User-Agent 可配置，默认使用真实浏览器 UA
- 请求间隔有下限保护（最低 1 秒）
- 仅用于个人学习用途，禁止商业使用和二次分发

## 4. 架构设计

### 4.1 模块划分

```
manga_scraper/
├── main.py                  # 入口，解析 CLI 参数
├── config.yaml              # 配置文件（含 cookie）
├── core/
│   ├── engine.py            # 核心调度引擎
│   ├── downloader.py        # 图片下载器（异步/多线程）
│   ├── storage.py           # 本地存储管理
│   └── db.py                # SQLite 状态管理
├── adapters/
│   ├── base.py              # 适配器基类（抽象接口）
│   └── site_xxx.py          # 具体网站适配器
├── models/
│   └── models.py            # 数据模型（Manga, Chapter, Page）
└── utils/
    ├── logger.py            # 日志工具
    ├── retry.py             # 重试装饰器
    ├── cookie_loader.py     # Cookie 加载与校验（配置文件/CLI/浏览器）
    └── http_client.py       # HTTP 客户端封装（自动挂载 Cookie、限速、重试）
```

### 4.2 适配器接口（Adapter Pattern）

```python
class BaseSiteAdapter(ABC):
    """网站适配器基类，每个目标网站实现此接口"""

    @abstractmethod
    def get_manga_list(self, mode: str, **kwargs) -> List[str]:
        """获取漫画列表 URL"""

    @abstractmethod
    def parse_manga_metadata(self, html: str) -> MangaMetadata:
        """解析漫画元数据"""

    @abstractmethod
    def parse_chapter_list(self, html: str) -> List[Chapter]:
        """解析章节列表"""

    @abstractmethod
    def parse_page_urls(self, html: str) -> List[str]:
        """解析章节内所有图片 URL"""

    @abstractmethod
    def get_image_url(self, page_url: str) -> str:
        """获取单页图片真实地址"""
```

### 4.3 核心流程

```
CLI 参数解析 → 加载配置 → 加载 Cookie → Cookie 有效性校验
    → 校验失败 → 提示用户更新 Cookie → 退出
    → 校验通过 → 创建 HTTP 客户端（自动挂载 Cookie）→ 选择适配器
    → 获取漫画列表（按指定模式）
    → 对每部漫画：
        → 解析元数据 → 创建目录 → 保存 metadata.json
        → 解析章节列表 → 过滤已下载
        → 对每个章节：
            → 解析图片列表 → 并发下载图片 → 更新数据库状态
    → 输出统计报告
```

**Cookie 加载优先级：**
1. 命令行参数 `--cookie`
2. 配置文件 `config.yaml` 中的 `cookie` 字段
3. 从本地浏览器自动提取（需安装 `browser-cookie3`，可选）

### 4.4 数据模型

```
Manga: id, title, author, cover_url, tags, updated_at, source_url
Chapter: id, manga_id, number, title, page_count, source_url
Page: id, chapter_id, page_num, url, file_path, size, status
```

## 5. 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| HTTP 客户端 | `httpx` | 支持异步，连接池，HTTP/2 |
| HTML 解析 | `BeautifulSoup4` + `lxml` | 容错性好，API 直观 |
| JSON 解析 | `jsonpath-ng` 或 `jmespath` | 解析 API 响应 |
| 并发 | `asyncio` + `aiofiles` | 异步 IO，高并发低开销 |
| 数据库 | `aiosqlite` | 轻量异步 SQLite |
| 配置 | `PyYAML` | YAML 可读性好 |
| CLI | `argparse` + `rich` | 美观的终端输出 |
| 限速 | `aiolimiter` | 异步令牌桶限速 |
| Cookie 提取 | `browser-cookie3`（可选） | 从浏览器自动读取 Cookie |

## 6. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 网站反爬（IP封禁） | 高 | User-Agent 伪装、请求间隔、代理池、验证码识别预留 |
| 图片链接失效/防盗链 | 中 | Referer 设置、重试机制、错误日志记录 |
| 页面结构变化 | 中 | 适配器独立，出问题只影响单站；解析加异常保护 |
| 大漫画下载时间长 | 低 | 断点续传、并发下载、增量模式 |
| 磁盘空间不足 | 低 | 下载前预估大小、可配置下载上限 |

**ID 规律**：chapterId = comicId 最后一位 hex + 1（如 `...87` → `...88`），后续章节 ID 需通过 API 获取。

### 8.5 图片 URL 构造

图片有两个访问路径：

**源站直连**（需 Referer/防盗链）：
```
{fileServer}/static/{path}
例如: https://storage-b.picacomic.com/static/tobeimg/BclOe4xxq48Gg-...
```

**代理加速**（Web 版使用）：
```
https://storage-b.go2778.com/static/tobs/{uuid}.jpg
例如: https://storage-b.go2778.com/static/tobs/f3ffa34f-c3ee-4656-8913-a2512f87973c.jpg
```

### 8.6 阅读器类型

Web 版使用 `waterfall`（瀑布流）阅读模式，所有图片加载在同一页面上下滚动。
